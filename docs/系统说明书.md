# 应急普法系统说明书

> 项目名称：Emergency Law（应急普法系统）  
> 适用对象：毕业设计答辩、指导老师评审、系统使用与维护人员

---

## 1. 项目概述

### 1.1 项目背景
本系统面向“应急普法”场景，目标是提供一个集 **普法课程学习、在线测试、经验分享、后台管理** 于一体的 Web 平台，帮助用户高效学习应急相关法律知识，并支持管理员进行课程、题库、用户和社区内容运营。

### 1.2 建设目标
- 为普通用户提供清晰的学习路径：课程 → 测试 → 分享。
- 为管理员提供统一的管理控制台，降低运营成本。
- 支持本地视频上传与在线内容混合教学。
- 实现基本权限控制与内容审核流程。

### 1.3 系统特色
- 前后端分离架构，便于扩展与部署。
- 课程支持文本、链接、视频三种课时类型。
- 支持经验分享、评论、点赞、防重复点赞。
- 管理端可进行帖子审核并查看详情。
- 兼容多种密码格式（明文/MD5/BCrypt）以适配历史数据。

---

## 2. 系统技术架构

### 2.1 总体架构
系统采用 B/S 架构，分为：
- 前端：Vue + Vite + Element Plus
- 后端：Spring Boot + MyBatis-Plus
- 数据层：MySQL（核心业务数据）+ JSON 文件（部分轻量互动数据）

### 2.2 技术栈
- **后端**：Java 8+, Spring Boot 2.7.x, MyBatis-Plus, JWT
- **前端**：Vue 3, Vite, Axios, Element Plus
- **数据库**：MySQL 8.x（推荐）
- **构建工具**：Maven、npm

### 2.3 运行端口（默认）
- 后端：`8080`
- 前端：`5173`
- MySQL：`3306`

## 3. 功能模块说明

## 3.1 用户端模块

### 3.1.1 登录与鉴权
- 用户通过账号密码登录。
- 登录成功后下发 Token，前端后续请求携带 Authorization。
- 路由根据角色进行页面访问控制。

### 3.1.2 普法学习
- 课程列表浏览、分类筛选。
- 课程详情查看，支持课时学习与进度保存。
- 课时内容类型：
  - `TEXT`：文本教学内容
  - `LINK`：外部学习链接
  - `VIDEO`：视频地址（支持本地上传后地址）

### 3.1.3 在线测试
- 用户可进行题目练习/试卷作答。
- 可查看测试历史与成绩信息（按系统接口支持范围）。

### 3.1.4 经验分享
- 发布经验帖子（标题、内容、标签）。
- 点赞功能（同一用户对同一帖子不可重复点赞）。
- 评论功能（查看评论列表、发表评论）。

---

## 3.2 管理端模块

### 3.2.1 运营总览
展示课程、试卷、帖子、测验记录等核心统计指标，辅助运营决策。

### 3.2.2 课程管理
- 新建/编辑/发布/归档课程。
- 课时管理：新增、编辑、删除。
- 视频课时支持：
  - 填写视频链接；
  - 上传本地视频到服务器后自动保存 URL。

### 3.2.3 题库与试卷管理
- 试卷创建、编辑、发布、归档。
- 题目管理（新增/编辑/删除）。
- 支持规则配置（及格分、时限、重考、难度占比等）。

### 3.2.4 帖子审核
- 审核分享帖子（通过/驳回）。
- 支持标签维度过滤。
- 支持查看帖子详细内容，辅助审核判断。

### 3.2.5 用户管理
- 用户列表查看。
- 角色调整、启用/停用、密码重置。
- 管理员可退出并回到登录页。

---

## 4. 核心业务流程

### 4.1 学习流程
用户登录 → 进入课程列表 → 选择课程与课时 → 学习并保存进度 → 继续下一课时。

### 4.2 测试流程
用户进入测试模块 → 开始答题 → 提交试卷 → 查看成绩与历史。

### 4.3 经验分享流程
用户发布帖子 → 帖子进入审核/展示链路 → 其他用户点赞与评论互动。

### 4.4 管理审核流程
管理员进入帖子审核 → 查看详情与标签 → 审核通过/驳回 → 前台状态同步。

---

## 5. 数据与存储说明

### 5.1 MySQL 主要业务数据
- 用户、课程、课时、试卷、题目、学习进度、测试记录等结构化数据。

### 5.2 文件存储
- 本地上传视频保存在 `data/uploads/...`。
- 通过后端静态资源映射 `/uploads/**` 对外访问。

### 5.3 轻量互动数据
- 部分分享模块交互数据（如标签/评论/点赞）采用 JSON 文件存储（位于 `data/` 下）。

---

## 6. 安全与权限设计

### 6.1 登录安全
- 使用 Token 机制进行接口访问鉴权。
- 前端请求统一注入 Authorization 头。

### 6.2 角色权限
- 普通用户：学习、测试、分享。
- 管理员：进入管理端，具备课程/题库/用户/审核管理权限。

### 6.3 密码兼容策略
系统密码匹配支持：
- 明文
- MD5（含 `{MD5}` 前缀形式）
- BCrypt

用于兼容历史账号迁移场景。

## 7. 部署与启动说明（摘要）

### 7.1 环境要求
- JDK 8+
- Maven 3.8+
- Node.js 18+
- MySQL 8.x

### 7.2 启动步骤
1. 配置数据库连接（`src/main/resources/application.yml`）。
2. 在项目根目录启动后端：`mvn spring-boot:run`。
3. 在 `emergency-law-web` 启动前端：`npm install && npm run dev -- --host`。
4. 浏览器访问：`http://localhost:5173`。

> 详细配置请参考项目 `README.md` 中“详细环境配置与启动方式”章节。

## 8. 常见问题与处理建议

### 8.1 前端代理报错 `ECONNREFUSED`
原因：前端可启动，但后端 `8080` 未启动或不可达。  
处理：先确认后端健康接口 `http://localhost:8080/health`。

### 8.2 上传本地视频报 `Network Error`
可能原因：后端未启动、上传接口不可达、超时或文件过大。  
处理：检查后端状态、网络请求、上传限制配置。

### 8.3 数据库连接异常
可使用仓库脚本快速排查：

```bash
bash scripts/check-db-connection.sh
```
